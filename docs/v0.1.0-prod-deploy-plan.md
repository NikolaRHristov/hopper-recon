# Production deployment plan — v0.1.0

**Status:** shipped (with one deferral — see "Update")
**Date:** 2026-05-07 · **Updated:** 2026-05-09

## Update — 2026-05-09

**GHCR image publishing is intentionally deferred.** Hopper Recon at v0.1 is consumed only by its operator, who builds locally with `docker compose up -d --build`. Multi-arch GHCR + cosign + SBOM is overhead we don't need yet. The release workflow that did this is deleted in commit `599bd20` but preserved in git history — bring it back when we have external consumers pulling pinned images. The CI workflow (checks-only — gofmt / vet / tsc / eslint / vitest) ships and runs on every push.

The reference `docker-compose.yml` below is correct, but the `image:` lines now refer to locally-built images (`hopper-recon:latest`, `hopper-recon-web:latest`) rather than `ghcr.io/...:0.1.0`. Operators use `docker compose up -d --build` rather than pulling. Everything else in this doc still holds.

## Position

For v0.1.0 OSS, **a high-quality `docker-compose.yml` plus clear documentation is enough** for most production deploys. We do **not** ship a Helm chart, Kustomize overlay, NetworkPolicy template, or ServiceMonitor in v0.1.0.

Rationale: mature devops teams have opinionated infrastructure (their own helm conventions, GitOps patterns, secret managers, ingress controllers). They prefer a working container image and clear interfaces over vendor-shipped charts they have to fork. Smaller teams run docker-compose directly on a single VM. Both groups are well-served by lean primitives.

## What v0.1.0 ships (mandatory)

### Container side

1. **Multi-arch images on GHCR** — `linux/amd64` + `linux/arm64`, free, no Docker Hub rate limits
2. **Semver tags** per release: `v0.1.0`, `v0.1`, `v0`, `latest`
3. **Reference `docker-compose.yml`** at repo root that runs end-to-end with `docker compose up`
4. **`/healthz`** + **`/readyz`** endpoints on engine and web (`/api/healthz` for the Next route)
5. **Structured JSON logs to stdout** — operators put their preferred log shipper in front
6. **Non-root user, no privileges** — `USER 1000:1000` in Dockerfile, no `CAP_*` adds
7. **Graceful shutdown** — both engine and web handle SIGTERM cleanly

### Documentation side

8. **`DEPLOY.md`** at repo root, covering:
   - Every env var (name, default, what it controls, secret-or-config marker)
   - Every port (and why each is needed)
   - Every volume (path, contents, persistence required y/n)
   - Resource expectations: `~250 MiB RAM idle, ~700 MiB during scan; 100m CPU idle, 1 vCPU during scan`
   - Auth posture: **no built-in auth in v0.1** — put behind VPN/Tailscale/oauth2-proxy/Cloudflare Access; v0.3 ships Auth.js
   - Backup recipe: `sqlite3 /data/scans.db .backup /backup/scans-$(date +%F).db`
   - Upgrade recipe: pull new image, `docker compose up -d`; schema migrations auto-run on engine boot
9. **Release notes per tag** in GitHub Releases — what changed, schema migrations, breaking notes

That is the entire mandatory list.

## Reference `docker-compose.yml`

```yaml
services:
  engine:
    image: ghcr.io/iksnerd/hopper-recon:0.1.0
    restart: unless-stopped
    user: "1000:1000"
    volumes:
      - ./data:/data                                              # SQLite persistence
      - ~/.config/subfinder:/root/.config/subfinder:ro            # OSINT API keys (optional)
      - ~/.config/uncover:/root/.config/uncover:ro                # OSINT API keys (optional)
      - ~/.config/hopper-recon:/root/.config/hopper-recon:ro      # GeoLite2 mmdb (optional)
    healthcheck:
      test: ["CMD", "/healthz"]
      interval: 10s
      timeout: 3s
      retries: 3

  web:
    image: ghcr.io/iksnerd/hopper-recon-web:0.1.0
    restart: unless-stopped
    user: "1000:1000"
    depends_on:
      engine:
        condition: service_healthy
    environment:
      - ENGINE_URL=http://engine:8080
      - HOPPER_DB_MODE=engine
    ports:
      - "9120:9120"
    healthcheck:
      test: ["CMD", "wget", "-qO-", "http://127.0.0.1:9120/api/healthz"]
      interval: 10s
      timeout: 3s
      retries: 3
```

This file lives at the repo root and serves as both the trial / dev experience (`docker compose up`) and the canonical reference topology that devops translates into their own deployment tooling.

## Deploy patterns operators will pick from (without our help)

| Pattern | Who picks it | What they do |
|---|---|---|
| Single VM + docker-compose + systemd | small/mid teams | rent a VM (Hetzner CX11, Lightsail, DO Droplet — $5–20/mo); systemd unit runs `docker compose up`; nginx or Caddy reverse-proxies; restic/borg backs up the SQLite volume nightly |
| k8s + their own Deployment manifest | platform teams with templates | derive a ~30-line Deployment from compose; integrate with existing ingress / cert-manager / secrets infrastructure |
| k8s + their own Helm chart | shops that template everything | same, wrapped in a chart that fits their internal conventions |
| Nomad jobspec | HashiCorp shops | translate compose to HCL |
| ECS / Fly.io / Railway / Render | "managed platform" preference | port compose to platform-specific config; many of these accept compose directly |

Shipping a Helm chart actively *harms* patterns 2/3/4/5 — those teams either ignore it (now we maintain a chart no one uses) or have to fork it (now they document the divergence). Letting them write their own is the correct call.

## What we DON'T ship in v0.1.0

| Excluded | Why |
|---|---|
| Helm chart | Mature shops write their own; small shops use compose. Add later if 5+ people file the issue. |
| Kustomize overlays | Same reasoning. |
| NetworkPolicy template | Provider-specific (Calico vs Cilium vs the cloud's CNI); devops writes their own. |
| ServiceMonitor / Prometheus rules | Their cluster, their metrics conventions. We expose `/metrics` and stop. |
| Air-gapped install guide | "Mirror the image to your registry" is a one-liner; doesn't need a guide. |
| Backup CronJob template | `sqlite3 .backup` in a `kubectl exec` is shorter than templatizing it. |
| Operator / CRDs | Probably never. K8s ecosystem is allergic to gratuitous CRDs. |
